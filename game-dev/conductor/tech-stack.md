# Tech Stack

## Primary Languages
- **Luau:** The foundational, optimized language for native Roblox development.
- **TypeScript (via Rojo/Roblox-TS):** Used for advanced tooling, strong typing, and seamless monorepo integration.
- **Python / Node.js:** For server-side analysis, agentic system integration, and automation scripts outside the Roblox environment.
- **Open Strategy:** The project remains open to incorporating other languages (e.g., Rust, Go) as the monorepo evolves and new needs arise.

## Frameworks and Libraries
- **Roact / React Lua:** For building complex, declarative user interfaces within Roblox.
- **Fusion / Flamework:** Modern, reactive frameworks for state management and boilerplate reduction.
- **Custom Evolving Frameworks:** A commitment to building and iterating on internal frameworks that align with the "dnp" philosophy.

## Infrastructure and Tooling
- **Rojo:** The essential tool for syncing local file structures with Roblox Studio, enabling professional workflow and version control.
- **Bun:** The primary runtime and package manager for the monorepo, prioritizing speed and efficiency.
- **PostgreSQL / Redis:** For external data persistence, server analysis, and state management outside of Roblox.
- **Sovereign Tooling:** Prioritizing tools that allow for local control and minimize dependence on proprietary, closed-source ecosystems where possible.

## Evolution and Research Strategy
- **Non-Crystallized Architecture:** Systems are designed to be modular and replaceable, avoiding technical debt from rigid dependencies.
- **Conductor-Driven Research:** Utilizing the AI agent (Conductor) to continuously research and propose new technologies, frameworks, and patterns as the platform and ecosystem evolve.
- **Documentation-First:** Every technological shift is documented to maintain the project's history and ensure philosophical alignment.
